================================================================================
1. SET UP CREDENTIALS
================================================================================

Step 1: Copy the environment template
  $ cp .env.example .env

Step 2: Edit .env file and add your PostgreSQL credentials:
  DB_HOST=localhost
  DB_PORT=5432
  DB_NAME=railway_db
  DB_USER=postgres
  DB_PASSWORD=your_password_here

  Optional: Add Firebase credentials for cloud backup features
  FIREBASE_CREDENTIALS_PATH=firebase-credentials.json

================================================================================
2. SET UP DATABASE
================================================================================

Option A: Using the setup script (Linux/Mac)
  $ bash setup_db.sh

Option B: Manual setup
  1. Create database: createdb -U postgres railway_db
  2. Import schema:
     psql -U postgres -d railway_db -f database/schema.sql
  3. Import seed data:
     psql -U postgres -d railway_db -f database/seed.sql
  4. Import views, functions, procedures, triggers:
     psql -U postgres -d railway_db -f database/views_procedures.sql

================================================================================
3. RUN THE APPLICATION
================================================================================

Activate virtual environment:
  $ source zenv/bin/activate

Start the Flask development server:
  $ python run.py

The application will be available at:
  http://localhost:5000

Default dashboard: http://localhost:5000/dashboard

================================================================================
8. TROUBLESHOOTING
================================================================================

Q: "Database connection failed"
A: Check your .env file has correct PostgreSQL credentials

Q: "Module not found"
A: Run: pip install -r requirements.txt

Q: "Port 5000 already in use"
A: Change port in run.py or kill existing process

Q: "Firebase errors"
A: Cloud features are optional. Set FIREBASE_CREDENTIALS_PATH in .env
